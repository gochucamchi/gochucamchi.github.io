void start() 함수 처음에 1번만 실행되는 함수

주로 변수의 시작값을 지정하기 위해서 쓰였다.

Update() 프레임 마다 한번씩 호출되는 함수이다. 60프레임이 나오면 60번이 호출이 된다는 뜻이다.

죽음 상태가 아니라면 점프 벽타기 착지등이 가능하게 만들었다.

FixedUpdate() Update와는 다르게 Fixed Timestep이라는 내가 스스로 설정한 값에 따라서 일정시간단위로 스스로 호출된다.

일정하게 호출이 되다보니 단발성이 아닌 물리효과에서 많이 사용이 된다. AddForce(Impulse)의 경우가 단발성으로 힘을 주는거니 이런 경우는 제외

Landing() 벽과 닿지않고 땅과도 닿아있지 않다면 애니메이션의 플레이어 점프라는 변수를 트루로 만들고 닿아있다면 false로 만듦

Jump() 땅에 닿아있는 상태에서 스페이스바를 누르면 점프 사운드와 함께 rigidbody의 rigid에 위쪽 방향으로 JumpPower 만큼의 힘으로 단발성으로 힘을 주어라는 명령어이다.

Walljump() 벽과 닿아있으며 땅과는 닿아있지 않은 상태일 때 스페이스바를 누르면 플레이어가 보는 방향의 반대로 힘을 가해 점프하게 만들었다.

그 외로는 가속도를 없애고 벽점프의 변수가 true로 바뀌게 했다.

Wallwait()땅과 닿아있지 않지 않으며 벽에 붙어있으면 wallwait이라는 변수가 true로 바뀐다

부가적으로는 벽에 닿아있는 동안 중력이 약해지며 벽에 붙은 애니메이션이 실행이 되며 캐릭터의 방향을 쿼터니어를 이용해 180도 틀어버린다.

만약 위 조건이 틀리다면 wallwait은 false가 되고 중력은 다시 정상으로 돌아오고 플레이어 애니변수는 false가 된다.

Movement() 이게 은근히 난해하게 만들어놨다

벽점프를 하지 않으며 벽에 붙어있지 않은상태에서 플레이어가 맞지도 않은상태에서 오른쪽 방향키를 누르면 그 방향으로 이동하며 방향을 전환하는데

이때 left_right_change변수는 오른쪽 키보드를 누른걸 인식하고 저장한다. 그리고 벽점프를 위해서 벡터값을 저장한다.

당연히 반대방향에서의 벽점프 구현을 위해 반대 방향에도 잘 넣었다.

IsOnWall() 옆으로 raycastHit를 쏴서 Ground라는 레이어와 부딪혔는지 확인한다. 이후 값을 반환한다

// 옆으로 세워놓은 벽도 ground로 만들었다

IsOnGround() 바닥으로 raycastHit를 쏴서 Ground라는 레이어와 부딪혔는지 확인한다. 이후 값을 반환한다

OnDrawGizmos() 레이저의 범위가 잘 되었는지 가시적인 확인이 필요하여 썼다.

나머지는 적들과 히트관련이거나 사망관련 혹은 아이템을 먹었을때등의 처리를 했다.

            this.transform.position = Vector3.Lerp(this.transform.position, targetPosition, moveSpeed * Time.deltaTime);            
 
            float x = Mathf.Clamp(transform.position.x, xMin, xMax);
            float y = Mathf.Clamp(transform.position.y, yMin, yMax);
 
        transform.position = new Vector3(x, y, -10);
        }
        Camera_range();
    }
